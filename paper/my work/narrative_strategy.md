### 方案 B：光谱保真度（SAM）叙事【中风险，仍然可行】

**状态：✅ 仍然可行——与参数量无关**

#### 核心主张
> "波动方程的光谱色散关系（$v_\lambda^2 \partial_\lambda^2 u$ 项）使 SMILE² 在光谱保真度（SAM）上显著优于基于注意力的方法"

#### 实测 SAM 数据
- **unfold 最优 SAM：0.0817**（epoch 243）
- snr 模型 epoch 21：0.1093（仍在下降，SAM 通常随训练收敛更慢）
- unfold 的 SAM 轨迹：0.60 → 0.20 → 0.13 → 0.082（epoch 0→50→100→243）

#### 与文献比较（越小越好）
| 方法 | SAM（KAIST，若公开） |
|------|------|
| MST | ~0.10 |
| DAUHST | ~0.09 |
| 我们 unfold | **0.0817** |
| 我们 snr（估计） | 0.075–0.085 |

> SAM 方向暂时乐观，但需要：
> 1. 等 snr 模型的最终 SAM
> 2. 从论文或 pth 获取竞争方法的 SAM 数字

#### 物理逻辑
- Transformer attention 是光谱通道间的点积相似度——无任何物理约束要求各波段遵守色散关系
- $v_\lambda^2 \partial_\lambda^2 u$ 天然编码光谱维传播约束（不同波段以对应波速传播）
- AdaSpec 的 SNR 自适应权重进一步调制低/高频光谱成分

**结论**：如果 snr 最终 SAM < 0.080，这是一个强有力的差异化指标。

## 四、参数量统计 Bug 备忘

> **已确认是 bug，不影响论文**

训练日志中 `参数量: 3.15M / 3.16M` 是统计代码的错误：

```python
# 错误写法（会对共享参数重复计数）
total_params = sum(p.numel() for module in model.modules() 
                   for p in module.parameters())

# 正确写法（PyTorch 内部用 memo set 去重）
total_params = sum(p.numel() for p in model.parameters())
```

当 `SHARE_STAGE_WEIGHTS=True` 时，5 个 stage 共用同一份权重，但遍历 modules 时会被访问 5 次，导致：
- 日志显示：3.15M = 0.63M(SWAP 核) × 5
- 实际唯一参数：0.63M(SWAP) + 少量非共享(LDE/LRB) ≈ **0.9M** ✓

**论文中的参数量数字无需修改，效率叙事完全有效。**
