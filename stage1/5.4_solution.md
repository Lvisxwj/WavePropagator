# 5.4 Solution — WaveMST 代码改进方案

> 本文档只描述改动方案，不修改任何代码。

---

## 一、dataset.py — 新功能

### 1.0 整体流程（幂等检查优先）

`load_training()` 入口处**一次性**判断两个 JSON 是否都已存在：

```
info_path  = dataset/data_name/[data_name]_info.json
bands_path = dataset/data_name/[data_name]_bands.json

if info_path.exists() AND bands_path.exists():
    直接读取 bands_path 得到 num_bands，跳过全部扫描，正常加载并返回
else:
    执行加载循环，边读边累加统计，循环结束后写入两个 JSON
```

只要两个文件都在，就**完全跳过**扫描和写入，不做任何重复工作。

---

### 1.1 band 数软编码（来源：bands.json）

`load_training()` 结束后，调用方（train.py）从 `bands.json` 中读取 `num_bands` 字段，赋给训练全局变量 `NUM_BANDS`。dataset.py 内部所有硬编码 28 的地方改为使用该参数传入：

- `shift_back(inputs, step=2, nC)` — nC 必须显式传入，不再自行假设 28
- `load_test(test_path, nC)` — 同上，line 185 的 `== 28` 改为 `== nC`
- `_splice_four(train_data, crop_half, nC)` — line 222 `torch.zeros(28,...)` 改为 `torch.zeros(nC,...)`

train.py 在 `main()` 中 `load_training()` 返回后立即读取 `bands.json` 获得 `NUM_BANDS`，之后所有调用都传入该值，不再使用 CONFIG 里的硬编码默认值作为依据。

---

### 1.2 生成 `[data_name]_info.json`（加载循环中内联统计）

**位置**：`dataset/[data_name]/[data_name]_info.json`，例如  
`dataset/CAVE_1024/CAVE_1024_info.json`

**内容示例**
```json
{
  "data_name": "CAVE_1024",
  "num_scenes": 205,
  "num_bands": 28,
  "scenes": [
    {
      "filename": "scene_001.npy",
      "wavelength_min_nm": 453,
      "wavelength_max_nm": 681
    }
  ]
}
```

**实现**：在加载循环内，每成功读取一个文件时，顺手记录该文件的 `num_bands`（即 `img.shape[-1]`）和波长范围（从 physics.py 的 `WAVELENGTHS` 取首尾值）；循环结束后统一写入，**无需第二次遍历**。

---

### 1.3 生成 `[data_name]_bands.json`（加载循环中内联累加）

**位置**：`dataset/[data_name]/[data_name]_bands.json`，例如  
`dataset/CAVE_1024/CAVE_1024_bands.json`

**内容示例**
```json
{
  "data_name": "CAVE_1024",
  "num_bands": 28,
  "wavelengths_nm": [453, 457, 462, ...],
  "band_mean": [0.312, 0.318, ...]
}
```

**实现**：在加载循环内维护一个累加器 `band_sum = np.zeros(nC)`，每读取一张图执行 `band_sum += img.mean(axis=(0,1))`（img 为 [H, W, nC]）；循环结束后除以场景总数得到 `band_mean`，连同 `WAVELENGTHS` 一起写入。**同一次循环内同时完成 info 和 bands 两个 json 的数据收集**，循环结束后各写一次文件。

---

## 二、physics.py — WAVELENGTHS 软编码

### 2.1 重命名常量

将 `WAVELENGTHS_CAVE_28` → `WAVELENGTHS`（全局搜索替换，同时更新所有引用该名称的文件：physics.py 本身，以及任何 import 它的模型文件：wpo3d_phys.py、helm_pure.py、wpo3d_helm.py 等）。

### 2.2 从 bands.json 动态读取

physics.py 顶部不再硬编码波长列表，改为接收一个 `bands_json_path` 参数（或由 train.py 在 import 后调用初始化函数），逻辑如下：

1. 接收 `bands_json_path`（由 train.py 根据 `DATA_NAME` 拼接传入，格式为 `dataset/[data_name]/[data_name]_bands.json`）。
2. 读取 `wavelengths_nm` 字段填充模块级变量 `WAVELENGTHS`。
3. 若路径不存在，fallback 到原有硬编码列表并打印警告，保持向后兼容。

**json 路径不能在 physics.py 内部硬编码**，必须由 train.py 根据当前使用的数据集名称动态传入，这样换数据集时 physics.py 无需修改。

**注意**：`get_k_phys_for_dim` 和 `get_inv_lambda_for_dim` 内部的 `num_bands` 默认值从 `28` 改为 `len(WAVELENGTHS)`，实现完全软编码。

## 三、train.py — JSON 驱动参数 + mask 选项 + 时间打印

### 3.0 从 bands.json 读取 NUM_BANDS

在 `main()` 中，`load_training()` 调用完成后：
1. 读取 `dataset/[data_name]/[data_name]_bands.json`，取 `num_bands` 字段覆盖 `NUM_BANDS`。
2. 将 `bands_json_path` 传给 physics.py 的初始化函数，确保 `WAVELENGTHS` 与数据集一致。
3. 后续所有涉及 nC / NUM_BANDS 的调用（`shift_back`、`load_test`、`_splice_four`、模型构建）均使用此值。

CONFIG 区域的 `NUM_BANDS = 28` 保留作 fallback 默认值，正常情况下会被 json 值覆盖。

---

### 3.1 保留 mask 选项

目前 `MASK_MODE = 'A'` 已在 CONFIG 区域，`build_model()` 已将其传给支持 `mask_mode` 参数的模型（index 0/2/3）。

需要补充的：
- Model index 4 (`WaveMST_Phys`) 目前 `build_model` 未传 `mask_mode`，需补充。
- Model index 5/6 (`Helmholtzformer`/`WaveMST_Helm`) 同样检查是否有 `mask_mode` 参数，有则传入。
- 在 `train_epoch()` 中，`gen_meas` 用的 `INPUT_SETTING` 已是软编码，无需额外修改。

> **总原则**：`MASK_MODE` 始终从 CONFIG 读取，不在任何模型或函数内硬编码为 `'A'`。

### 3.2 训练时间打印

**格式**：`月.日.时:分`，24 小时制，例如 `5.4.2:19`

**新增辅助函数**（可放在 train.py 顶部）：
```
def _fmt_time(t: float) -> str:
    s = time.localtime(t)
    return f"{s.tm_mon}.{s.tm_mday}.{s.tm_hour}:{s.tm_min:02d}"
```

**打印位置与内容**

| 时机 | 打印内容 |
|------|----------|
| 第 1 epoch 开始前 | `训练开始: 5.4.2:19` |
| 每 50 epoch 时 | `[Epoch 050] 当前时间: 5.4.3:25  预计结束: 5.4.8:42` |

**预计结束时间计算**：
```
已用时 = 当前时间 - 训练开始时间
每 epoch 平均用时 = 已用时 / 当前 epoch
剩余 epoch = MAX_EPOCH - 当前 epoch
预计结束时间 = 当前时间 + 剩余 epoch × 每 epoch 平均用时
```

打印位置建议：在 `main()` 的 epoch 循环里，`epoch == 1` 前打印开始时间，`epoch % 50 == 0` 时在 checkpoint 保存代码旁打印时间。

## 四、loss.py — 频域指标

### 4.1 频域幅度差异（Spectral Amplitude Loss / Metric）

**函数名**：`freq_amplitude_loss(pred, gt)` / `torch_freq_amp_err(pred, gt)`

**计算方式**：
1. 对 `pred` 和 `gt` 各在空间维度 (H, W) 做 2D FFT，取幅度谱（`torch.fft.fftn` + `torch.abs`）。
2. 计算幅度谱的均方误差（MSE）或 L1 差，作为频域幅度损失/指标。
3. 对所有 band 取均值。

**用途**：辅助训练损失（可选加权）或评估时打印。

---

### 4.2 径向频率分布对比（Radially Averaged Power Spectral Density，RAPSD）

**函数名**：`torch_rapsd(img)` → 返回 1D 径向平均功率谱，shape `[R]`

**计算方式**：
1. 对 `[C, H, W]` 的图做 2D FFT 后取功率谱 `|F|²`。
2. 以频谱中心为圆心，按径向距离 r = 0, 1, 2, ... 分组，对每组内所有频率点的功率取均值，得到 RAPSD 向量。

**分层分析函数**：`torch_freq_band_err(pred, gt, low_ratio=0.1)` 

分低频/高频两段分别计算幅度差异：
- 低频区：径向频率 r < `low_ratio × min(H, W) / 2`
- 高频区：其余

返回字典 `{'low_freq_err': ..., 'high_freq_err': ..., 'total_freq_err': ...}`

---

### 4.3 在训练时打印

修改 `train_epoch()` 中的打印逻辑（目前只打印 loss）和 `test_epoch()` 中的打印逻辑（目前打印 PSNR/SSIM），改为：

```
[Epoch 050] Loss: 0.012345  Time: 42.3s  LR: 1.23e-04
         Test → PSNR: 29.84  SSIM: 0.8812  SAM: 0.0342
                FreqAmpErr: 0.00512  LowFreqErr: 0.00201  HighFreqErr: 0.00724
```

- `FreqAmpErr`、`LowFreqErr`、`HighFreqErr` 在 `test_epoch()` 中计算，与 PSNR/SSIM 同一轮打印。
- SAM 也顺带加入打印（`torch_sam` 已存在，train.py 目前未调用）。

## 五、viz.py — 频域可视化

### 5.1 新增函数

#### `show_freq_magnitude(img, title, save_path)`
显示图像的频域幅度谱（log 尺度，中心化显示）。

- 输入：`[C, H, W]`
- 对每个 band 做 2D FFT，取 `log(1 + |F_shifted|)`，展示均值或若干代表 band。
- 与 `show_bands` 风格一致，多 band 并排。

#### `show_freq_comparison(pred, gt, band, save_path)`
并排展示 pred 和 gt 在某个 band 的频域幅度谱，以及二者差值图（高频/低频差异一目了然）。

#### `show_rapsd(pred, gt, save_path)`
绘制 pred 和 gt 的径向平均功率谱密度曲线对比图（参照 loss.py 的 RAPSD 计算）。

### 5.2 合并到 `show_all()`

在 `show_all()` 末尾追加三个调用：

```python
# 6. 频域幅度图
show_freq_magnitude(p, title=f'Pred Freq Scene {scene_idx}',
                    save_path=str(save_dir / 'pred_freq.png'))
show_freq_magnitude(g, title=f'GT Freq Scene {scene_idx}',
                    save_path=str(save_dir / 'gt_freq.png'))

# 7. 频域并排对比（中间波段）
show_freq_comparison(p, g, band=mid,
                     save_path=str(save_dir / f'freq_compare_band{mid}.png'))

# 8. 径向功率谱对比
show_rapsd(p, g, save_path=str(save_dir / 'rapsd.png'))
```

### 5.3 保留用户 main

不删除或修改 viz.py 末尾的 `if __name__ == '__main__':` 块，只在 `show_all()` 函数体内追加，不影响独立运行。

## 六、其他建议（频域模型特色指标）

鉴于 WaveMST 是频域模型，以下额外指标/可视化有助于凸显模型特色和后续调试：

| 指标 / 可视化 | 说明 | 建议位置 |
|---|---|---|
| **频域 SSIM（F-SSIM）** | 在频域幅度谱上计算 SSIM，衡量纹理重建质量 | loss.py |
| **相位误差（Phase Error）** | FFT 后取相位，计算 pred/gt 相位差的均值，低则代表相位保真度好 | loss.py |
| **光谱功率比（Band Energy Ratio）** | `sum(|pred_fft|²) / sum(|gt_fft|²)` 逐 band，>1 过度增强，<1 模糊 | loss.py，test_epoch 打印 |
| **频域残差热图** | 频域幅度差 `|F(pred)| - |F(gt)|` 的空间热图（逆 FFT 回空域）| viz.py |
| **波段间频域相关性矩阵** | 不同 band 的频域幅度谱相关系数，诊断 band 间一致性 | viz.py（可选） |

这些改动不需要修改模型结构，只需在 loss.py 和 viz.py 增加函数，并在 test_epoch 和 show_all 中调用。
